# 医学事务基础框架

## 医学事务部（Medical Affairs）定位

连接研发、注册、市场、销售、临床、KOL的枢纽部门，核心职责：

| 职能 | 说明 |
|------|------|
| 学术支持 | 为内部团队提供医学专业知识 |
| 医学沟通 | 与KOL/学会/医疗机构建立学术联系 |
| 证据生成 | 真实世界研究、研究者发起研究（IIT） |
| 医学信息 | 文献追踪、竞争情报、医学问答 |
| 合规审核 | 确保所有对外材料医学合规 |

## 医学经理日常工作流

### 每日
- 追踪产品线最新文献（PubMed Alerts）
- 回复医学咨询（内部+外部）
- 跟进在研IIT/RWE项目

### 每周
- KOL拜访与维护
- 内部分享会/培训
- 竞品动态更新

### 每月
- 科室会/学术会议材料更新
- 月度学术推广报告
- 销售代表医学培训

### 每季度
- 季度学术推广策略复盘
- 最新临床数据整合
- 产品知识更新手册

## 合规红线（中国区）

- 禁止超适应症推广（违反《药品管理法》）
- 禁止绝对化用语（"最""第一""突破性"等需有证据）
- 禁止贬低竞品（可呈现客观数据对比，不可主观评价）
- 所有推广材料需标注数据来源和引用
- 学术赞助需合规备案
- IIT需伦理委员会批准

## AI辅助价值

| 场景 | AI价值 | 注意事项 |

| KOL画像 | 文献关联分析 | 需保护隐私 |
| 竞争分析 | 多维度竞品对比 | 数据时效性需确认 |
